# 系统第一阶段局限性说明 (Limitations)

本项目目前处于 **第一阶段（骨架与最小可行版本验证）**，当前版本的核心作用是确立系统边界、数据校验标准和节点协作链路。在商业应用或实际开发中存在以下限制：

## 1. 算法与接口限制
- **全 Mock 机制**：系统当前并未接入真实的外部大语言模型 API（如 Gemini、GPT-4、Claude 等）。所有的 Agent 推理过程均为在本地预设好的规则匹配与 Mock 数据流转。
- **静态 RAG 检索**：`retriever.py` 从本地的 `reference_works.json` 静态文件中进行匹配，而非连接真实的向量数据库（如 Milvus, Chroma）或搜索引擎，不支持海量自然语言泛化检索。

## 2. 状态与存储限制
- **内存级别 Memory**：项目记忆 (`project_memory.py`) 和角色记忆 (`character_memory.py`) 均使用纯 Python dict 在内存中存储。这意味着如果后端 API 服务重启，所有的多轮记忆和设定记录都将被清空。
- **无持久化数据库**：没有接入外部关系型数据库（如 PostgreSQL）或文档数据库（如 MongoDB），不支持大规模的并发用户、权限隔离与历史数据持久化。

## 3. 并发与安全限制
- **单用户原型**：目前未设计多租户权限校验，所有的多轮记忆直接通过 `project_title` 为 Key 进行管理，无法隔离多个不同用户的并发请求。
- **本地环境运行**：目前尚未进行生产级别的安全防护、限流及防注入攻击配置。
